```python
class Solution:
    def solveNQueens(self, n: int) -> List[List[str]]:
        ans = []
        queens = [0] * n
        col = [False] * n
        diag1 = [False] * (n * 2 - 1)
        diag2 = [False] * (n * 2 - 1)

        def dfs(r: int) -> None:
            if r == n:
                ans.append(['.' * c + 'Q' + '.' * (n - 1 - c) for c in queens])
                return
            
            for c, ok in enumerate(col):
                if not ok and not diag1[r + c] and not diag2[r - c]:
                    queens[r] = c
                    col[c] = diag1[r + c] = diag2[r - c] = True
                    dfs(r + 1)
                    col[c] = diag1[r + c] = diag2[r - c] = False
        
        dfs(0)
        return ans
```
本质上是豪华版全排列：枚举列号的全排列
`queens` -> 相当于 path，`queens[r] = c` 记录皇后的位置（放皇后）
`col` 和 `diag1` 和 `diag2` -> 帮助判断其他皇后的攻击范围（当前皇后会不会被攻击到）
遍历每个列的位置，判断能不能放皇后，放完皇后更新三个辅助数组的布尔值，继续 `dfs(r + 1)`。记得回溯

如何判断斜方向皇后的攻击范围？对于 ↗ 方向的格子，行号加列号是不变的，对于 ↖ 方向的格子，行号减列号是不变的。一个方向上共有 `2 * n - 1` 个斜方向
